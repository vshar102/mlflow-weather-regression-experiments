# MLflow-Tracked Weather Regression Experiments

`mlflow-weather-regression-experiments`  ·  Portfolio project  ·  MIT License

## Overview

Runs and tracks multiple regression-model experiments predicting mean temperature from London weather data, using MLflow for experiment tracking rather than ad-hoc notebook comparisons.

## Problem Statement

Comparing multiple model configurations (e.g. different regressors, hyperparameters) is common in applied ML, but without experiment tracking it's easy to lose track of which run produced which metric. This project demonstrates a proper MLflow-tracked workflow.

## Approach

Trains several scikit-learn regression models (e.g. decision tree, random forest) on London weather data, logging parameters, metrics, and model artifacts to MLflow for each run so results are comparable and reproducible.

## Tech Stack

Python, scikit-learn, MLflow, pandas

## Results

See `mlflow ui` (instructions below) for a comparable table of runs, parameters, and validation metrics across the experiments logged in this project.

## Setup

```bash
python -m venv .venv && source .venv/bin/activate
pip install scikit-learn mlflow pandas matplotlib seaborn jupyter
```

## Usage

```bash
jupyter notebook notebook.ipynb   # run experiments
mlflow ui                          # then open http://localhost:5000 to compare runs
```

## Security Considerations

No API keys or credentials are committed to this repository. Where the project
calls an external API, copy `.env.example` to `.env` and supply your own key —
`.env` is git-ignored.

## Troubleshooting

- If a notebook cell fails on a missing package, install the pinned versions
  in `requirements.txt` (or the imports listed in Tech Stack above) rather than
  the latest release, since some ML libraries change APIs between minor versions.
- Large datasets and model checkpoints are intentionally excluded from this
  repository (see `.gitignore`) to keep it lightweight — see Setup for how to
  obtain or regenerate them.
